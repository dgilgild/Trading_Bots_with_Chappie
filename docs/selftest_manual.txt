Self-test manual

Purpose
- Validate strategy logic and backtest execution quickly after code changes.
- Catch regressions in signals, trade counts, and PnL baselines.

What is included
- scripts/test_strategies_selftest.py
  - Compatibility wrapper that runs pytest suite under tests/strategies
  - Prints legacy-style summary: Total / Passed / Failed / Skipped
  - Keeps non-zero exit code when pytest reports failures

How to run
1) Install dependencies
   pip install -r requirements.txt

2) Run self-tests
   PYTHONPATH=. python3 scripts/test_strategies_selftest.py
   - Internally executes pytest tests/strategies -q with JUnit XML output.
   - If pytest is missing, the wrapper prints install steps and exits non-zero.

2a) Run pytest by marker (tests/strategies)
   PYTHONPATH=. pytest -m unit tests/strategies -q
   PYTHONPATH=. pytest -m smoke tests/strategies -q
   PYTHONPATH=. pytest -m regression tests/strategies -q

2b) Run pytest suite and export CSV in one command
   PYTHONPATH=. python3 scripts/run_strategy_tests_and_export_csv.py

   Output files:
   - docs/test_reports/strategies_pytest_results.xml
   - docs/test_reports/strategies_pytest_results.csv
   - docs/test_reports/history/strategies_pytest_results_<UTC_TIMESTAMP>.xml
   - docs/test_reports/history/strategies_pytest_results_<UTC_TIMESTAMP>.csv

   Notes:
   - The non-history files are always overwritten with the latest run.
   - Timestamped copies are preserved in docs/test_reports/history.

2c) Run walk-forward evaluator (ST-402)
    Deterministic synthetic mode:
    PYTHONPATH=. python3 scripts/run_walk_forward.py --mode deterministic --strategy ema_cross --train-pct 50 --test-pct 25 --step-pct 25

    Other strategy examples (same window policy, deterministic data):
    PYTHONPATH=. python3 scripts/run_walk_forward.py --mode deterministic --strategy donchian_breakout --train-pct 50 --test-pct 25 --step-pct 25
    PYTHONPATH=. python3 scripts/run_walk_forward.py --mode deterministic --strategy k_davey_mom_keltner --train-pct 50 --test-pct 25 --step-pct 25
    PYTHONPATH=. python3 scripts/run_walk_forward.py --mode deterministic --strategy basic_keltner_reversion --train-pct 50 --test-pct 25 --step-pct 25

    Supported strategies:
    - ema_cross
    - rsi_reversion
    - donchian_breakout
    - ema_trend_hold
    - bmsb
    - emalyarovich_smas
    - k_davey_mom_keltner
    - basic_keltner_reversion

    DB mode example:
    PYTHONPATH=. python3 scripts/run_walk_forward.py --mode db --strategy ema_cross --exchange binance --symbol BTC/USDT --timeframe 1d --start-date 2021-01-01 --end-date 2022-12-31 --train-pct 50 --test-pct 25 --step-pct 25

   Output files are written to:
   - docs/test_reports/walk_forward/<run_stamp>/walk_forward_windows.csv
   - docs/test_reports/walk_forward/<run_stamp>/walk_forward_summary.json

3) Read results
   - PASS means behavior matches expected baseline.
   - FAIL means behavior changed or broke.

Test groups explained
1) signal_logic / indicator_logic
   - Verifies core strategy conditions with controlled inputs.

2) backtest_smoke
   - Monkeypatches fetch_ohlcv with deterministic synthetic data.
   - Canonical source: src/core/testing/synthetic_data.py (make_synthetic_ohlcv_v1).
   - Ensures each run_backtest_* function returns expected output shape.

3) backtest_regression
   - Uses deterministic synthetic data and fixed parameters.
   - Validates generated trade CSV schema via src/core/testing/contracts.py.
   - Enforces timeline integrity in trade CSVs:
      - entry_time <= exit_time for each trade row
      - exit_time ordering is monotonically non-decreasing (strict by default)
   - Confirms per strategy:
      - total trade count
      - total net PnL (tolerance 1e-3)
      - first/last trigger text patterns in CSV

When to run
- Before committing changes to any strategy or backtester code.
- After modifying fees, stops, sizing, pyramiding, or entry/exit logic.
- After changing CSV schema fields used by reports/charts.

How to interpret failures
- signal_logic fail:
  Core strategy condition behavior changed; inspect strategy.py first.

- backtest_smoke fail:
  Runner API, imports, or execution flow is broken.

- backtest_regression fail:
  Either a real behavior change happened or baseline is stale.

Updating baselines intentionally
- If behavior changed by design, update expected values in:
  tests/strategies/conftest.py
  inside the regression_expected fixture dictionary.

Recommended workflow
1) Make strategy change.
2) Run self-tests.
3) If regression fails, verify if change is intentional.
4) Update baseline only for intentional changes.
5) Re-run until all tests pass.
