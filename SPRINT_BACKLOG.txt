Sprint Backlog - Backtesting System Hardening

Sprint objective
- Build a trustworthy validation pipeline: deterministic tests, baseline contracts,
  and strategy comparison artifacts.

Duration
- 2 weeks (10 working days)

Definition of done
- One command validates all strategies.
- Regression drift is immediately visible and actionable.
- Outputs are both machine-readable and human-readable.
- Docs explain run steps and failure interpretation.

Priority order
1) Test architecture and deterministic data contract
2) Schema/timeline contracts
3) Comparison artifacts
4) Walk-forward first slice
5) Final runbook and polish
